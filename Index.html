Git e GitHub

Comandos Git:


Comando Descrição
git init Marca a pasta onde será o repositório do git
git status Mostra o status do git
git config --local user.name "Seu nome aqui"
git config --local user.email "seu@email.aqui"		 Configuração de usuário
git add ./
git add "nome-arquivo" 		Adiciona os arquivos para o git, podendo ser um ou todos do repositório
git commit -m "Comentário commit" Salvas as informações das alterações
git log
git log --oneline git log -p 		Mostrar os logs, mais comandos em:
https://devhints.io/git-log
git remote add origin "Path Online" Configurar um caminho online
git push origin master Envia as requisições para o repositório
git branch branch-name Cria um novo branch
git checkout branch-name Começa a trabalhar no novo branch
git merge branch-name Realiza o merge entre branchs
git rebase branch-name Juntar os commits
git checkout -- nome-arquivo Cancelas as alterações que não foi feito o commit, voltando as últimas alterações
git reset HEAD nome-arquivo Cancela as alterações que foram adicionas para commit
git revert hash-commit Cancelar um commit específico
git stash Salvas as alterações temporariamente para depois
git stash list Mostra as alterações temporariamente guardadas
git stash pop Retira da lista de modificações e aplica
git diff Mosta as últimas alterações com as correntes
git diff hash..hash2 Mostra as diferenças entre 2 commits
git tag -a v0.1 -m "Primeira versão" Cria uma tag
git tag Mosta as tags
git push origin v0.1 Envia as tags para o repositório
git rebase -i [último commit] Juntando commits
git cherry-pick hash-commit Trazendo um commit específico
git bisect start
git bisect bad HEAD
git bisect good hash-commit 		Percorre a lista de commits até encontrar o resultado esperado
git show hash-commit Mostra todas as modificações do commit
git blame nome-arquivo Mostra o responsável para cada alteração de um arquivo
Issues

Reportar problemas, sugerir melhorias no código e organizar quaisquer coisas que façam sentido para o projeto.

Pull Requests

Através de pull requests, nós podemos enviar melhorias e correções para projetos.
Realizar um Fork para fazer as alterações.

Git Flow

Organização de branches que facilita a resolução de conflitos, inclusive para que bugs repetidos não aconteçam.

Git Cola, GitHub Desktop, GitKraken

Ferramenta de interface gráfica para termos algum auxílio durante o trabalho com o Git.
